## v13.2 — Claude API 4 分割

### `scripts/fetch_news.py` の書き換え (641 → 892 行)

旧: 1 回の Opus 呼び出しで 10 要素を一括生成
新: **4 つの呼び出しに分割、各々独立した try/except で失敗を吸収**

| Call | 担当要素 | モデル | max_tokens |
|------|---------|--------|-----------|
| `call_news_and_charts` | epigraph + headline + news(7) + charts_of_the_day | claude-opus-4-7 | 3500 |
| `call_deep_dive` | deep_dive (cadence 切替: 平日/週次/月次) | claude-opus-4-7 | 2500 |
| `call_cb_and_alts` | central_bank_watch + pe_pd_view + real_assets_view | claude-sonnet-4-6 | 2500 |
| `call_muse_and_economic` | funny_stories + economic_chart_of_the_day | claude-haiku-4-5 | 1500 |

### 主要設計
- **`_call_claude` ヘルパ**: 4 callers の共通処理 (msg.create + JSON 抽出 + コードブロック除去 + parse + ログ) を統合
- **`run_all_calls` orchestrator**: 4 つを sequential 実行、各 try/except で失敗を独立処理
- **`_FALLBACK_*` 定数**: 各 call が失敗した時の merge-able dict を 4 つ用意
  - 1 つ失敗しても他は生成される
  - 後続 (`attach_source_urls` / `normalize_chart_candidates`) は耐性あり
- **`build_news_list()`**: 各 call が共通の "[N] 付きニュース一覧" テキストを生成
- **`determine_cadence` は top-level に残置**: `tests/test_common.py` の import が壊れない (v13.2-05)

### JSON schema 不変性
出力 `data/news.json` のキー構成は v7 と完全に同じ → **フロント側の改修は一切不要**

### コスト削減効果 (見込み)
- Muse + Economic を Haiku 4.5 に切替で対象部分のコスト 1/15
- 月額 ~1,600 円 → ~1,000〜1,200 円見込み (要モニタ)

### 検証
- AST parse OK
- mock import で 18 関数全て確認
- 3 つのモデル定数を確認: `MODEL_OPUS = "claude-opus-4-7"` / `MODEL_SONNET = "claude-sonnet-4-6"` / `MODEL_HAIKU = "claude-haiku-4-5"`
- `determine_cadence` が top-level で公開されており、既存 pytest が通る
